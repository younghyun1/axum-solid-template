Technical Architecture Goals:
- Performance: all customer interactions (except signup) aimed at below 0.05ms; programming language with the fastest performance as well as the most correctness enforced being used
- Affordability: being performant means being able to serve more customers with the same amount of computational resources rented. this server should achieve ~20,000 RPS or more at 20 pounds a month.
- Maintainability: everything is documented in an agent-friendly manner for rapid but correct development; future developers will also benefit and business administration will also be able to implement features is the goal.
- Reliability: all data backed up regularly. automated redeployment, recovery procedures.
- Security: sensitive industry; no shortcuts taken (rust). everything will be encrypted and gated as much as possible. audits possible but expensive. photos and videos will not be
